# Deploying "We'll Share" to Render.com

The repo ships a **`render.yaml` blueprint** that provisions the entire stack on

## Free-tier behavior to expect

- **Cold starts:** free web services sleep after ~15 min idle. The first request
  after sleeping takes ~30â€“60 s while the backend wakes. Normal afterwards.
- **Uploads are ephemeral:** profile photos / document uploads written to
  `/app/uploads` are lost on restart (no persistent disk on free tier). For
  permanent storage, attach a Render disk (paid) or move uploads to S3/Cloudinary.
- **Free Postgres** has a storage cap and an expiry window on the free plan;
  fine for a demo. Upgrade the DB plan in `render.yaml` if you need it long-term.

## Troubleshooting

| Symptom | Fix |
|---|---|
| Backend log: `WeakKeyException` on boot | Set `JWT_SECRET` manually to a 32+ char string in the backend Environment tab. |
| Backend log: SSL error connecting to Postgres | Add `?sslmode=require` to the JDBC URL â€” set `DB_HOST` etc. stay, but override `spring.datasource.url` via an env var, or append in `application-prod.properties`. |
| Frontend loads but API calls fail (CORS) | Confirm the backend `FRONTEND_URL` env includes `https://*.onrender.com`. Already set by the blueprint. |
| Login works but no OTP email arrives | `MAIL_USERNAME` / `MAIL_PASSWORD` not set (expected if you left them blank). |
| `mvn`/Docker build OOM | Free tier builds on Render's build infra (not the 512 MB runtime), so this is rare; retry the deploy. |

## Updating the app later

Just push to `main`. `autoDeploy: true` is set on both services, so Render
rebuilds and redeploys automatically. No dashboard steps needed for updates.
